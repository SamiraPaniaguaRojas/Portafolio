Informe de Laboratorio 06 - Uso de Hooks en React

1. Temas aprendidos (definiciones y procedimiento)

useState: +
Hook que permite crear y manejar estados locales dentro de un componente funcional. Usado para variables que cambian y afectan la interfaz.
useEffect: +
Permite ejecutar efectos secundarios (como peticiones, timers o actualizar el DOM) cuando el componente se monta o cambian ciertas dependencias.
useContext: +
Para compartir datos globales sin tener que pasar props en múltiples niveles (contexto React).
useRef: +
Referencias persistentes que no causan re-render cuando cambian, útil para manipular elementos DOM o guardar valores mutables.
useReducer: +
Alternativa a useState para estados complejos con lógica más clara usando reductores (como Redux).
useCallback y useMemo: +
Optimizan el rendimiento evitando recrear funciones o cálculos complejos innecesariamente.
Hooks personalizados: +
Funciones que encapsulan lógica de hooks para reutilizar en varios componentes.

// Código React equivalente (simulado)
import React, { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <button onClick={() => setCount(count - 1)}>-</button>
      <span>{count}</span>
      <button onClick={() => setCount(count + 1)}>+</button>
    </div>
  );
}

3. Reflexión: ¿Qué aprendí? ¿Cómo lo aprendí?

Aprendí a manejar el estado y efectos secundarios en componentes funcionales con React Hooks, haciendo la UI más simple y reactiva. Entendí la importancia de separar lógica y presentación, optimizar rendimiento con useCallback y useMemo, y cómo los hooks personalizados mejoran la reutilización.

Aprendí principalmente mediante la práctica con pequeños proyectos, lectura de la documentación oficial y análisis de ejemplos reales en repositorios de GitHub.
