🎯 Eventos en JavaScript

Definición: Los eventos son acciones que ocurren en el navegador, como clics, teclas presionadas, o cambios en formularios. Son la base de la interactividad web.

Procedimiento:

// Agregar evento a un elemento element.addEventListener('click', function() { console.log('¡Elemento clickeado!'); }); // Evento con arrow function button.addEventListener('click', () => { // Código a ejecutar });

🔄 Renderizado Condicional

Definición: Es la técnica de mostrar u ocultar elementos del DOM basándose en condiciones específicas. Permite crear interfaces dinámicas que responden al estado de la aplicación.

Procedimiento:

// Mostrar/ocultar elementos if (condicion) { element.style.display = 'block'; } else { element.style.display = 'none'; } // Cambiar contenido dinámicamente element.innerHTML = condicion ? 'Texto A' : 'Texto B';

🔁 Renderizado Iterativo

Definición: Consiste en generar múltiples elementos HTML basándose en arrays o listas de datos. Es fundamental para mostrar listas dinámicas.

Procedimiento:

// Iterar sobre un array y crear elementos const datos = ['Item 1', 'Item 2', 'Item 3']; const container = document.getElementById('container'); datos.forEach(item => { const div = document.createElement('div'); div.textContent = item; container.appendChild(div); });

📝 Formularios en JavaScript

Definición: Los formularios permiten capturar y validar datos del usuario. JavaScript puede manejar eventos de formularios y manipular sus valores.

Procedimiento:

// Obtener valores de formulario const input = document.getElementById('miInput'); const valor = input.value; // Validar formulario form.addEventListener('submit', (e) => { e.preventDefault(); if (validarDatos()) { // Procesar formulario } });

🌐 Consumo de APIs

Definición: Las APIs (Application Programming Interfaces) permiten obtener datos de servicios externos. JavaScript puede consumir APIs usando fetch() o librerías como Axios.

Procedimiento:

// Con fetch (nativo) fetch('https://api.ejemplo.com/datos') .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error('Error:', error)); // Con async/await async function obtenerDatos() { try { const response = await fetch('https://api.ejemplo.com/datos'); const data = await response.json(); return data; } catch (error) { console.error('Error:', error); } }

🤔 Reflexión Personal

¿Qué aprendí?

Durante este laboratorio he consolidado conocimientos fundamentales sobre JavaScript moderno y desarrollo web interactivo. Específicamente aprendí a:

  • Manejar eventos del DOM de manera eficiente y estructurada
  • Implementar renderizado condicional para crear interfaces dinámicas
  • Utilizar renderizado iterativo para generar contenido basado en datos
  • Trabajar con formularios y validación de datos en tiempo real
  • Consumir APIs usando fetch() y manejar operaciones asíncronas
  • Aplicar promesas y async/await para código más limpio y mantenible
  • Integrar librerías externas como html2canvas para funcionalidades avanzadas

¿Cómo aprendí?

Mi proceso de aprendizaje se basó en una metodología práctica y progresiva:

  • Práctica dirigida: Implementé cada funcionalidad paso a paso, validando el funcionamiento antes de continuar
  • Investigación activa: Consulté documentación oficial de MDN y recursos especializados para entender conceptos profundos
  • Resolución de problemas: Enfrenté desafíos reales como la validación de datos y el manejo de estados de aplicación
  • Iteración y mejora: Refactoricé el código múltiples veces para optimizar rendimiento y legibilidad
  • Integración de tecnologías: Combiné HTML, CSS y JavaScript para crear una experiencia de usuario cohesiva
  • Testing manual: Probé exhaustivamente cada funcionalidad en diferentes escenarios de uso

Conceptos Clave Dominados

Los conceptos más importantes que he interiorizado incluyen:

  • Event-driven programming: Arquitectura basada en eventos para interfaces reactivas
  • DOM manipulation: Creación y modificación dinámica de elementos HTML
  • State management: Control del estado de la aplicación y sincronización con la UI
  • Asynchronous programming: Manejo de operaciones no bloqueantes y APIs externas
  • Error handling: Implementación robusta de manejo de errores y validaciones
  • Responsive design: Creación de interfaces adaptables a diferentes dispositivos
